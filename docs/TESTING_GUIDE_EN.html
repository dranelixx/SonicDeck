üéØWhat is Sonic Deck?

Sonic Deck is a desktop soundboard application built specifically for streamers, gamers, and content creators. The app allows you to play sounds simultaneously to two different audio outputs (e.g., your headphones AND your stream).

üéß Dual-Audio Routing

üîåSetting Up Virtual Audio Cable (For Streaming)

‚ñº

If you want to use Sonic Deck for streaming, you need a Virtual Audio Cable. This allows sounds to be routed to your stream/Discord/etc.

Step 1: Install VB-Audio Virtual Cable

‚ñº
  1. Download VB-Cable
    üì• Direct Download: VBCABLE_Driver_Pack45.zip
    Or visit: vb-audio.com/Cable (free, donation optional)
  2. Extract ZIP
    Right-click the ZIP ‚Üí "Extract All"
  3. Run Setup
    Right-click on VBCABLE_Setup_x64.exe ‚Üí "Run as Administrator"
  4. Complete Installation
    Click "Install Driver" ‚Üí Wait until "Installation completed" appears ‚Üí OK
  5. Restart PC (recommended)
    So the virtual device is properly recognized

Step 2: Open Windows 11 Sound Settings

‚ñº
üöÄ Quick Access with Win + R:
Press Win + R ‚Üí Type mmsys.cpl ‚Üí Enter
This opens the classic Sound settings directly!
‚ö†Ô∏è CRITICAL: Do NOT set "CABLE Input" as your default Playback device!
Keep your normal headphones/speakers as the default Windows playback device.
Alternatively (manual):
  1. Right-click the speaker icon in the taskbar
  2. Click "Sound settings"
  3. Scroll down to "Advanced"
  4. Click "More sound settings"
Show Screenshot (Windows Sound Settings)

Step 3: Set Up Microphone Routing (So they hear YOU + Sounds)

‚ñº

Since Discord will listen to the Cable (for sounds), we need to route your voice into that same Cable.

  1. Switch to the "Recording" tab
    Find your actual microphone (e.g., Logitech, Blue Yeti)
  2. Open Properties
    Right-click your microphone ‚Üí "Properties"
  3. Enable "Listen"
    Go to the "Listen" tab.
    Check the box "Listen to this device".
  4. Select Playback Device
    In the dropdown "Playback through this device", select "CABLE Input (VB-Audio Virtual Cable)".
    ‚úÖ Now your voice is sent into the cable along with the sounds!
    Show Screenshot (Microphone Listen)
  5. Click Apply

Step 4: Prepare CABLE Output

‚ñº
  1. Still in "Recording" tab
    Find "CABLE Output"
  2. Enable CABLE Output
    Right-click on "CABLE Output" ‚Üí "Enable"
    If not visible: Right-click in empty area ‚Üí "Show Disabled Devices"
    Show Screenshot (CABLE Output)
  3. Check levels
    Right-click ‚Üí "Properties" ‚Üí "Levels" tab ‚Üí Set volume to 100%

Step 5: Set Up in Sonic Deck

‚ñº
  1. Open Sonic Deck
    Go to Settings (gear icon in the sidebar)
  2. Select Monitor Output
    Choose your normal headphones/speakers
    This is what YOU will hear
  3. Select Broadcast Output
    Choose "CABLE Input (VB-Audio Virtual Cable)"
    This will be routed to your stream/Discord
  4. Play test sound
    Go back to Dashboard ‚Üí Play a sound ‚Üí You should hear it AND OBS should detect it
    Show Screenshot (Sonic Deck Settings)

Step 6: Set Up in Discord

‚ñº
üéÆ Discord Audio Setup:
  1. Open Discord ‚Üí Settings (gear icon bottom left)
  2. In the left menu: "Voice & Video"
  3. Input Device: Select "CABLE Output (VB-Audio Virtual Cable)"

    Show Screenshot (Discord Devices)
üìã Additional Settings (on the same page):

    Show Screenshot (Voice Processing)
  • Noise Reduction: "Standard" is OK

‚úÖ Summary:
‚Ä¢ Input = Your Microphone
‚Ä¢ Output = CABLE (not your headphones!)
‚Ä¢ Echo Cancellation = OFF
‚Ä¢ Krisp = None/Standard
Test:
  1. Join a voice channel
  2. Play a sound in Sonic Deck
  3. Your Discord friends should hear the sound! üéâ

Troubleshooting Virtual Cable

  • CABLE Output not visible: Restart PC after installation
  • Discord hears nothing: Check if CABLE Output is set as default recording device
  • Distorted sound: Set levels in Windows to 100%, then lower volume in Sonic Deck
  • Echo in Discord: Your microphone is picking up sounds from your headphones - lower volume!
  • Others hear themselves: This is normal if you play their audio through Sonic Deck

Edge Cases & Finding Issues

üîç Please test these scenarios:

  • Very large audio files (>10 MB)
  • Very short sounds (<1 second)
  • Fast spam-clicking on sounds
  • Device switching during playback
  • Minimize/maximize app during playback
  • Play many sounds in parallel (10+)
  • Sound library with 50+ sounds

‚ö†Ô∏èKnown Limitations

‚ñº
  • Spam-Clicking: Very fast repeated clicking (>10x/second) may cause sounds to not play
    ‚Üí Wait briefly between clicks (~100ms) for reliable playback
  • No Hotkeys: Global hotkeys are not yet implemented (planned for Phase 5)
  • No System Tray: App must remain in foreground
  • No Autostart: Must be started manually
  • Windows SmartScreen: Warning appears because app is not signed

üí° Why no Spam-Clicking?

The app starts a separate audio thread for each sound. With extreme spam-clicking (e.g., 20 clicks in 1 second), the system cannot react fast enough and sounds will be skipped. This is a technical limitation of the current architecture.

Normal fast clicking (2-5x/second) works perfectly! Only extreme button-mashing is problematic.

üîßTroubleshooting

‚ñº

App won't start

  • Check if antivirus is blocking the .exe
  • Run setup as administrator
  • Add Windows Defender exception

No sound

  • Check Settings ‚Üí are the correct devices selected?
  • Is the volume at 0%?
  • Are your audio devices enabled in Windows?
  • Test with a simple sound (e.g., short beep)

Performance issues

  • Cache is managed automatically (500MB limit)
  • With many sounds: Check Task Manager ‚Üí RAM usage
  • Large files are decoded in the background

Finding Logs & Configuration Files

Location (Windows): %LOCALAPPDATA%\com.sonicdeck.app\

Log Files: logs\sonicdeck.YYYY-MM-DD.log
- Example: sonicdeck.2025-12-20.log
- Automatic daily rotation (last 7 days are kept)
- Contains errors, warnings, and information about app activities
- Opens directly with double-click in Notepad/Editor

Configuration:
- settings.json (Your settings)
- sounds.json (Sound library)

3. Copy the newest .log file (with today's date) and send with bug report
